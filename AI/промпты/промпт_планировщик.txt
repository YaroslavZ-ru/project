================================================================================

1.1. Прочитай финальное описание проекта:
     Файл в папке AI с "финал" или "описание_проекта_финал" в названии.
     Из него ты поймёшь:
     - Что такое AI-Terminator и какова его цель
     - Архитектуру системы: все модули, их роли и связи

     - Какие файлы должны существовать в итоге и что делать каждый

1.2. Прочитай ВСЕ планы с датой в названии (это финальные версии):
     Файлы вида план_1-5_12.06.2026.txt, план_6-10_12.06.2026.txt и т.д.
     Из них ты поймёшь:
     - Какие изменения уже запланированы или выполнены
     - Нумерацию изменений (чтобы продолжить с правильного номера)
     - Стиль и формат написания плана — СКОПИРУЙ ЭТОТ ФОРМАТ ТОЧНО
     - Какие модули уже созданы к каждому этапу
     - Принципы проекта (Python 3.10+, pathlib, logging и т.д.)

1.3. Посмотри также планы без даты (план_20-25.txt, план_26-30.txt и т.д.):
     Они могут содержать идеи и наброски для будущих этапов.
     Используй их как источник идей, но не копируй слепо —
     финальное описание важнее. Если есть противоречие — приоритет у описания.

1.4. Посмотри реальный код проекта (папки src/, scripts/, tests/):
     Убедись что понимаешь что уже реально написано.
     Это важно: план должен строиться на том что есть, а не на том что должно быть.

ШАГ 3. НАПИШИ НОВЫЙ ПЛАН
================================================================================

3.1. ФОРМАТ ФАЙЛА — строго как в существующих планах:

     Начало файла (обязательно):
     -----------------------------------------------------------------------
     ОБЯЗАТЕЛЬНО ДЛЯ ИИ: ВСЁ ЧТО НАПИСАНО В ЭТОМ ФАЙЛЕ — РЕКОМЕНДАЦИИ, А НЕ ПРИКАЗЫ.
     ПРИМЕРЫ КОДА — ТОЛЬКО ИЛЛЮСТРАЦИИ НАМЕРЕНИЯ, НЕ КОПИРОВАТЬ БУКВАЛЬНО.
     ПИШИТЕ КАК ЛУЧШЕ, ЕСЛИ ЗНАЕТЕ. ГЛАВНОЕ — СОБЛЮДАТЬ ОПИСАННОЕ ПОВЕДЕНИЕ И СТРУКТУРУ.
     -----------------------------------------------------------------------

     Заголовок:
     ================================================================================
     План реализации AI-Terminator — Изменения NN-MM
     ФИНАЛЬНАЯ ВЕРСИЯ ДД.ММ.ГГГГ
     ================================================================================

     Блок "ЧТО УЖЕ ЕСТЬ" — перечисли все модули и их ключевые сущности
     которые существуют на момент начала этого плана. Формат:
     ================================================================================
     ЧТО УЖЕ ЕСТЬ (после изменений 1-XX)
     ================================================================================
       src/config.py       — Config (поле1, поле2, ...)
       src/модуль.py       — КлассИлиФункция (что делает)
       ...

     Блок "ЧТО СОЗДАЁМ" — список файлов которые появятся в этом плане:
     СОЗДАВАЕМЫЕ ФАЙЛЫ:
       src/новый_модуль.py    (Изменение NN)
       tests/test_модуль.py
       ...

     ИЗМЕНЯЕМЫЕ ФАЙЛЫ (если есть):
       src/config.py          (N новых полей)
       configs/config.json    (N новых полей)
       ...

     Для каждого изменения — отдельный блок:
     --------------------------------------------------------------------------------
     ИЗМЕНЕНИЕ NN: Название
     --------------------------------------------------------------------------------

     ЧТО НУЖНО СДЕЛАТЬ
     -----------------
     [Подробное описание что именно создать или изменить]

     ПОЧЕМУ ЭТО НУЖНО
     ----------------
     [Зачем этот модуль в архитектуре, как он связан с остальными]

     ДЕТАЛИ РЕАЛИЗАЦИИ
     -----------------
     [Список требований, алгоритм, поведение при ошибках, крайние случаи]
     [Примеры кода как ИЛЛЮСТРАЦИИ — не для копирования]

     ИНТЕРФЕЙС (публичные методы/функции)
     -------------------------------------
     [Сигнатуры функций с type hints и описанием что возвращают]

     ТЕСТЫ
     -----
     [Что именно проверить в тестах, конкретные assert-проверки]

     ПРОВЕРКА РАБОТОСПОСОБНОСТИ
     --------------------------
     [Как убедиться что изменение работает: запуск, вывод в консоль]

3.2. ТРЕБОВАНИЯ К СОДЕРЖАНИЮ:

     - Каждое изменение описывай исчерпывающе, без пробелов
     - Упоминай ВСЕ крайние случаи: None, пустая строка, файл не найден,
       сеть недоступна, данных нет в БД, модель не загружена
     - Указывай зависимости: "этот модуль использует Config из src/config.py"
     - Прописывай поля конфига которые добавляются или используются
     - Описывай логирование: что именно логировать (DEBUG/INFO/WARNING/ERROR)
     - Для каждого нового поля конфига: тип, значение по умолчанию, назначение
     - Для каждого нового файла данных: формат, структура, пример содержимого
     - Не оставляй ничего на "понять самостоятельно" — пиши явно

3.3. СТИЛЬ:
     - Русский язык, деловой тон
     - Разделители из "=" и "-" как в существующих планах
     - Нумерация изменений сквозная (продолжение от последнего)
     - Примеры кода оформляй в отступах, помечай "ИЛЛЮСТРАЦИЯ:"

================================================================================
ШАГ 4. НАЗВАНИЕ И СОХРАНЕНИЕ ФАЙЛА
================================================================================

4.1. Имя файла формируется по шаблону:
     план_NN-MM_ДД.ММ.ГГГГ.txt

     Где:
     - NN — номер первого изменения в этом плане
     - MM — номер последнего изменения
     - ДД.ММ.ГГГГ — текущая дата

     Пример: план_20-25_14.06.2026.txt

4.2. Сохрани файл в папку AI/ проекта в кодировке UTF-8.

4.3. После сохранения выведи:
     - Имя созданного файла
     - Список изменений которые вошли в план (номер + название)
     - Краткое (2-3 строки) описание что будет реализовано в этом блоке

ОБЩИЕ ПРАВИЛА
================================================================================

- Финальное описание проекта — главный источник правды
- Если планы без даты противоречат финальному описанию — доверяй описанию
- Не дублируй то что уже есть в предыдущих планах с датой
- Пиши план так чтобы другой ИИ мог выполнить его без дополнительных вопросов
- Мельчайшие детали важны: лучше написать лишнее чем упустить нужное

  не читая других изменений этого же плана
