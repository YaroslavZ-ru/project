ЧТО НУЖНО СДЕЛАТЬ
-----------------
Создать эталонный датасет data/eval_dataset.json и скрипт scripts/evaluate.py,
который вычисляет метрики качества: Precision@5 и Context Accuracy.

ПОЧЕМУ ЭТО НУЖНО
----------------
Без объективной оценки качества невозможно понять деградировала ли система
после изменений. evaluate.py запускает пайплайн на фиксированном наборе
примеров и сравнивает выдачу с ожидаемыми параметрами. Precision@5 показывает
долю релевантных параметров среди топ-5 предложенных. Context Accuracy — долю
запросов, для которых определён правильный домен.
Оба файла указаны в финальном описании проекта.

ФОРМАТ data/eval_dataset.json
------------------------------
Список объектов. Каждый объект — один тест-кейс:
  {
    "id":       "eval_001",
    "term":     "ключ гаечный",
    "hints":    ["техника", "вращение"],
    "expected_domain": "слесарный инструмент",
    "expected_params": ["size_mm", "material", "torque_nm"]
  }

  Поля:
    id              — уникальный идентификатор (строка)
    term            — термин для анализа (непустая строка)
    hints           — список уточнений (может быть пустым)
    expected_domain — ожидаемый домен из selected_context.domain
                      (если null — домен не проверяется)
    expected_params — список name-ключей параметров, которые ДОЛЖНЫ попасть
                      в топ-5 ответа (порядок не важен)

Минимальный датасет (не менее 6 кейсов):
  eval_001: "ключ гаечный",     ["техника"],          "слесарный инструмент",
            ["size_mm", "material"]
  eval_002: "ключ разводной",   ["инструмент"],       "слесарный инструмент",
            ["size_range_mm", "material"]
  eval_003: "ключ скрипичный",  ["музыка"],           "музыка",
            ["clef_type", "staff_position"]
  eval_004: "ключ",             [],                   null,
            []   (пустые expected_params — тест только на статус ok)
  eval_005: "болт",             ["крепёж"],           null,
            []
  eval_006: "гайка",            ["металл", "резьба"], null,
            []

Файл создать ВРУЧНУЮ (это данные, не генерируемый код). Структура JSON —
корректный UTF-8 с отступами 2 пробела.

ДЕТАЛИ РЕАЛИЗАЦИИ scripts/evaluate.py
---------------------------------------
Импорты (stdlib): json, logging, sys, argparse, time
from pathlib import Path

Опциональный импорт компонентов проекта (внутри __main__ или в теле функций):
from src.config import Config
from main import _init_components, run_pipeline  (или дублировать логику)

ВАЖНО: evaluate.py запускает ПОЛНЫЙ run_pipeline для каждого кейса.
Если FastText недоступен — векторы будут нулевыми, поиск не сработает,
но тест fallback-пути тоже полезен.

Dataclass EvalResult:
  id:              str
  term:            str
  status:          str      ("ok", "error")
  returned_params: list[str]  (имена параметров name из ответа)
  returned_domain: str | None
  expected_params: list[str]
  expected_domain: str | None
  precision_at_5:  float    (вычисляется методом)
  domain_correct:  bool
  duration_s:      float

  Метод compute_precision_at_5(self) -> float:
    top5 = self.returned_params[:5]
    Если expected_params пустой: return 1.0 (нет требований -> считаем ok)
    hits = len(set(top5) & set(self.expected_params))
    return hits / min(5, len(self.expected_params))

  Метод compute_domain_correct(self) -> bool:
    Если self.expected_domain is None: return True (домен не проверяем)
    return self.returned_domain == self.expected_domain

Функция run_evaluation(dataset_path: Path, config: Config,
                        components: tuple) -> list[EvalResult]:
  Args:
    dataset_path: путь к eval_dataset.json
    config:       Config
    components:   результат _init_components(config) — tuple из 7 элементов
  Returns:
    Список EvalResult для каждого кейса.

  Алгоритм:
    1. Загрузить dataset_path (json.loads).
       При ошибке: logger.error, return [].
    2. Распаковать components:
         (synonym_dict, lemmatizer, embedding_model, vector_cache,
          kb, generative_expander, session_manager) = components
    3. Для каждого кейса:
       a. Зафиксировать start = time.monotonic().
       b. Вызвать run_pipeline:
            result = run_pipeline(
                term=case["term"],
                hints=case.get("hints", []),
                debug=False,
                min_confidence=None,
                cfg=config,
                lemmatizer=lemmatizer,
                synonym_dict=synonym_dict,
                embedding_model=embedding_model,
                vector_cache=vector_cache,
                kb=kb,
                generative_expander=generative_expander,
                session_manager=session_manager,
                session_id=None,
            )
       c. duration = time.monotonic() - start
       d. Извлечь returned_params:
            [p["name"] for p in result.get("parameters", [])]
       e. Извлечь returned_domain:
            result.get("selected_context", {}).get("domain")
       f. Создать EvalResult, вычислить precision_at_5 и domain_correct.
       g. logger.info("eval %s: P@5=%.2f domain=%s t=%.3fs",
                      case["id"], er.precision_at_5, er.domain_correct, duration)
    4. Вернуть список EvalResult.

Функция print_report(results: list[EvalResult]) -> None:
  Вывести в stdout таблицу:
    ID       | Term          | P@5  | Domain | Time
    ---------|---------------|------|--------|------
    eval_001 | ключ гаечный  | 1.00 | OK     | 0.123s
    ...
  Итоговая строка:
    "Precision@5 (avg): X.XX | Context Accuracy: X.XX | Total: N кейсов"
  Если N == 0: вывести предупреждение.

Функция compute_summary(results: list[EvalResult]) -> dict:
  avg_precision = среднее precision_at_5 по всем кейсам
  context_accuracy = доля кейсов где domain_correct == True
  return {
      "precision_at_5":    round(avg_precision, 4),
      "context_accuracy":  round(context_accuracy, 4),
      "total_cases":       len(results),
      "ok_cases":          len([r for r in results if r.status == "ok"]),
      "error_cases":       len([r for r in results if r.status == "error"]),
      "avg_duration_s":    round(sum(r.duration_s for r in results) / max(1, len(results)), 4),
  }

Блок if __name__ == "__main__":
  Условный sys.path.insert.
  argparse:
    --dataset  путь к eval_dataset.json (default="data/eval_dataset.json")
    --config   путь к конфигу (default="configs/config.json")
    --json     флаг: вывести итог в JSON-формате вместо таблицы
  Загрузить Config, вызвать _init_components.
  Вызвать run_evaluation, print_report (или json-вывод), compute_summary.
  Вывести summary в stdout.

КРАЙНИЕ СЛУЧАИ:
  - Файл eval_dataset.json не найден: вывести ERROR, sys.exit(1).
  - Пустой датасет: вывести WARNING, вывести нулевые метрики.
  - run_pipeline бросил исключение: поймать, записать EvalResult со status="error",
    precision_at_5=0.0, domain_correct=False; продолжить.
  - FastText не загружен: результаты будут fallback-ответами; это валидный сценарий.

ДЕТАЛИ РЕАЛИЗАЦИИ scripts/profile.py
--------------------------------------
Импорты (stdlib): logging, sys, argparse, time, statistics
from pathlib import Path

Тестовые запросы (встроенные по умолчанию):

      {"term": "ключ",         "hints": ["техника"]},

Функция profile_pipeline(config: Config, components: tuple,
                          queries: list[dict],
                          n_runs: int = 3) -> dict:
  Args:
    config:     Config
    components: tuple из _init_components
    queries:    список запросов {"term": ..., "hints": ...}
    n_runs:     сколько прогонов каждого запроса
  Returns:
    Словарь со статистикой: {имя_шага: {"min_s", "max_s", "mean_s", "calls"}}

  Алгоритм:
    1. Создать словарь timings: dict[str, list[float]] для накопления замеров.
       Ключи: "total", "preprocess", "vectorize", "search", "aggregate_or_fallback"
    2. Для каждого запроса, для каждого прогона:
       a. Зафиксировать start_total = time.monotonic().

       b. ШАГ 1 — preprocess:
            t0 = time.monotonic()
            processed = preprocess(term, hints, cfg, synonym_dict, lemmatizer)
            timings["preprocess"].append(time.monotonic() - t0)

       c. ШАГ 2 — vectorize (с кэшем):
            t0 = time.monotonic()
            query_vector = vector_cache.get(term, hints, cfg)
            if query_vector is None:
                query_vector = vectorize(processed, embedding_model)
                vector_cache.put(term, hints, cfg, query_vector)
            timings["vectorize"].append(time.monotonic() - t0)

       d. ШАГ 3 — search:
            t0 = time.monotonic()
            candidates = kb.search_similar_concepts(
                query_vector, max_candidates=cfg.max_candidates)
            timings["search"].append(time.monotonic() - t0)

       e. ШАГ 4-5 — aggregate или fallback:
            t0 = time.monotonic()
            if candidates:
                aggregate_parameters(candidates, [], cfg.max_parameters)
            else:
                fallback_response(term, processed, cfg)
            timings["aggregate_or_fallback"].append(time.monotonic() - t0)

       f. timings["total"].append(time.monotonic() - start_total)

    3. Вычислить для каждого ключа: min, max, mean, count.
    4. Вернуть stats.

ИЗМЕНЕНИЕ 32: scripts/build_faiss.py — построение FAISS-индекса
================================================================================

ЧТО НУЖНО СДЕЛАТЬ
-----------------
Создать скрипт scripts/build_faiss.py, который читает все эмбеддинги концептов
из БД, строит FAISS-индекс и сохраняет его на диск. Сохранённый индекс
загружается KnowledgeBase при use_faiss=True и len(concepts) > faiss_threshold.

ПОЧЕМУ ЭТО НУЖНО
----------------
При большой базе знаний (тысячи концептов) линейный поиск через numpy
становится медленным. FAISS обеспечивает O(log N) поиск. Скрипт нужен
для предварительного построения индекса перед деплоем с use_faiss=True.
scripts/build_faiss.py указан в финальном описании проекта.

ДЕТАЛИ РЕАЛИЗАЦИИ scripts/build_faiss.py
------------------------------------------
На уровне модуля:
  _FAISS_AVAILABLE = True
  try:
      import faiss
  except ImportError:
      _FAISS_AVAILABLE = False

Функция build_faiss_index(config: Config) -> dict:
  Args: config — Config
  Returns:
    {"index_path": str, "vectors_count": int} при успехе.
    {"error": str} при критической ошибке.

  Алгоритм:
    1. Если not _FAISS_AVAILABLE:
         logger.error("faiss-cpu не установлен. pip install faiss-cpu")
         return {"error": "faiss_not_installed"}

    2. Определить output_path:
         faiss_path_str = config.faiss_index_path
         Если faiss_path_str (непустой):
           output_path = Path(faiss_path_str)
         Иначе:
           output_path = Path(config.db_path).parent / "faiss.index"
           logger.warning("faiss_index_path не задан, сохраняю в %s", output_path)

    3. Загрузить концепты из БД:
         from src.knowledge_base import KnowledgeBase
         kb = KnowledgeBase(config=config, embedding_model=None, synonym_dict=None)
         concepts = kb.get_all_concepts(use_cache=False)
         kb.close()
         Если concepts пустой:
           logger.error("База концептов пуста. Сначала выполните seed_data или update_kb.")
           return {"error": "empty_database"}

    4. Проверить что у концептов есть ненулевые эмбеддинги:
         valid = [c for c in concepts if not np.all(c["embedding"] == 0)]
         Если len(valid) < len(concepts):
           logger.warning("Пропущено %d концептов с нулевым эмбеддингом",
                          len(concepts) - len(valid))
         Если valid пустой:
           logger.error("Нет концептов с ненулевым эмбеддингом. Запустите seed_data.")
           return {"error": "no_valid_embeddings"}

    5. Построить матрицу эмбеддингов:
         matrix = np.stack([c["embedding"] for c in valid]).astype(np.float32)
         norms = np.linalg.norm(matrix, axis=1, keepdims=True)
         matrix = matrix / np.maximum(norms, 1e-9)  (нормализовать для cosine)

    6. Построить индекс:
         dim = matrix.shape[1]  (должно быть 300)
         index = faiss.IndexFlatIP(dim)  (Inner Product = cosine для нормализованных)
         index.add(matrix)
         logger.info("FAISS индекс построен: %d векторов, размерность %d",
                     index.ntotal, dim)

    7. Сохранить маппинг concept_id -> индекс (нужен для поиска):
         id_map_path = output_path.with_suffix(".ids.json")
         id_map = [c["id"] for c in valid]
         id_map_path.write_text(json.dumps(id_map, ensure_ascii=False), encoding="utf-8")

    8. Сохранить индекс:
         output_path.parent.mkdir(parents=True, exist_ok=True)
         faiss.write_index(index, str(output_path))
         logger.info("Индекс сохранён: %s", output_path)
         logger.info("Маппинг ID сохранён: %s", id_map_path)

    9. Вывести напоминание:
         Установить в configs/config.json:
           "use_faiss": true
           "faiss_index_path": "{output_path}"

   10. Вернуть {"index_path": str(output_path), "vectors_count": index.ntotal}

  При Exception: logger.error, return {"error": str(exc)}.

ПОЧЕМУ ЭТО НУЖНО
----------------
Текущий data/synonyms.json — пустой заглушечный файл или содержит минимум данных.
Настоящий словарь синонимов строится из RuWordNet (ruwordnet.ru) — русскоязычного
аналога WordNet с синсетами, гиперонимами и гипонимами. Без полного словаря
синонимов веса в формуле векторизации (0.1 * синонимы) не работают.
scripts/build_synonyms.py указан в финальном описании проекта.

ДЕТАЛИ РЕАЛИЗАЦИИ scripts/build_synonyms.py
---------------------------------------------
На уровне модуля:
  _RUWORDNET_AVAILABLE = True
  try:
      import ruwordnet
  except ImportError:
      _RUWORDNET_AVAILABLE = False

ФОРМАТ ВЫХОДНОГО data/synonyms.json:
  Словарь: {лемма: список синонимов}
  Пример:
    {
      "ключ": ["отмычка", "пароль", "тональность", "родник"],
      "болт": ["шуруп", "крепёж", "стержень"],
      "гайка": ["контргайка", "крепёж"]
    }
  Все слова в нижнем регистре, лемматизированы.
  Синонимы — список строк (леммы).
  Максимум 10 синонимов на слово (избыточные отрезать).

Функция build_from_ruwordnet(output_path: Path,
                              lemmatizer,
                              max_synonyms: int = 10) -> dict:
  Args:
    output_path:  куда сохранить synonyms.json
    lemmatizer:   Lemmatizer (для лемматизации слов из RuWordNet)
    max_synonyms: максимум синонимов на слово
  Returns:
    {"words_count": N, "synonyms_avg": float, "saved_to": str} или {"error": str}

  Алгоритм:
    1. Если not _RUWORDNET_AVAILABLE:
         logger.error("ruwordnet не установлен. pip install ruwordnet")
         return {"error": "ruwordnet_not_installed"}

    2. Загрузить RuWordNet:
         wn = ruwordnet.RuWordNet()
         logger.info("RuWordNet загружен")

    3. Собрать синонимы:
         synonyms_dict: dict[str, set[str]] = {}
         Для каждого синсета (sense group) в wn:
           members = все леммы синсета (приведённые к нижнему регистру)
           members_lemmas = [lemmatizer.lemmatize_word(m) for m in members]
           members_lemmas = [m for m in members_lemmas if m]  (убрать пустые)
           Для каждого слова w в members_lemmas:
             Добавить в synonyms_dict[w] все остальные слова синсета.

         ПРИМЕЧАНИЕ: RuWordNet API может отличаться от версии к версии.
         Проверить актуальный API: ruwordnet.RuWordNet() и методы итерации синсетов.
         ИЛЛЮСТРАЦИЯ:
           for synset in wn.synsets():
               senses = [s.lemma.lower() for s in synset.senses]
               senses_lemmas = unique_ordered([lem.lemmatize_word(s) for s in senses])
               for word in senses_lemmas:
                   if word not in synonyms_dict:
                       synonyms_dict[word] = set()
                   synonyms_dict[word].update(
                       s for s in senses_lemmas if s != word
                   )

    4. Ограничить количество синонимов:
         result = {w: list(syns)[:max_synonyms]
                   for w, syns in synonyms_dict.items()
                   if syns}

    5. Сохранить:
         output_path.parent.mkdir(parents=True, exist_ok=True)
         output_path.write_text(
             json.dumps(result, ensure_ascii=False, indent=2),
             encoding="utf-8"
         )

    6. Статистика:
         avg_syns = sum(len(v) for v in result.values()) / max(1, len(result))
         logger.info("Словарь синонимов: %d слов, avg %.1f синонимов", len(result), avg_syns)
         return {"words_count": len(result), "synonyms_avg": round(avg_syns, 2),
                 "saved_to": str(output_path)}

  При Exception: logger.error, return {"error": str(exc)}.
