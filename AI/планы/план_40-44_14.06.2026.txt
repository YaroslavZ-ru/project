ИЗМЕНЕНИЕ 40: scripts/healthcheck.py — диагностика всех компонентов
================================================================================

ЧТО НУЖНО СДЕЛАТЬ
-----------------
Создать скрипт scripts/healthcheck.py, который проверяет работоспособность
каждого компонента проекта и выводит читаемый отчёт. Выходной код 0 если
все критичные компоненты в порядке, 1 если есть проблемы.

ПОЧЕМУ ЭТО НУЖНО
----------------
При развёртывании на новой машине разработчик не знает что работает, а что нет:
есть ли FastText модель, доступна ли БД, правильно ли заполнен конфиг,
работают ли все опциональные зависимости. healthcheck.py даёт мгновенный
ответ на все эти вопросы за один запуск без изучения кода.

ДЕТАЛИ РЕАЛИЗАЦИИ scripts/healthcheck.py
------------------------------------------
Импорты: logging, sys, json, sqlite3, argparse
from pathlib import Path

Результат каждой проверки — именованный tuple или dataclass:
  CheckResult(name: str, status: str, detail: str)
  status: "OK", "WARN", "FAIL"

Функция check_config(config_path: Path) -> CheckResult:
  Попытаться загрузить Config.from_json(config_path, ...).
  OK:   конфиг загружен, все поля прошли валидацию.
  FAIL: FileNotFoundError или ValueError из _validate.
  Detail: если OK — f"db_path={cfg.db_path}, min_confidence={cfg.min_confidence}"
          если FAIL — текст ошибки.

Функция check_database(config) -> CheckResult:
  Проверить: Path(config.db_path).exists().
  Если не существует: FAIL, "Файл БД не найден: {path}"
  Подключиться sqlite3.connect, выполнить "SELECT COUNT(*) FROM concepts".
  OK:   "{N} концептов в базе"
  WARN: 0 концептов — "БД пустая, запустите scripts/seed_data"
  FAIL: sqlite3.Error — текст ошибки.

Функция check_fasttext(config) -> CheckResult:
  model_path = Path(config.fasttext_model_path)
  Если not model_path.exists():
    WARN: "Модель не найдена: {path}. Поиск будет работать через fallback."
  Иначе:
    Попытаться from src.embeddings import FastTextWrapper;
    wrapper = FastTextWrapper(model_path=model_path, fallback_path=None, cache_size=10)
    vec = wrapper.get_word_vector("тест")
    Если not np.all(vec == 0): OK "Загружена, dim={wrapper.get_dimension()}"
    Иначе: WARN "Модель загружена но вектор нулевой"
  При Exception: FAIL текст исключения.

Функция check_fallback_embeddings(config) -> CheckResult:
  fb_path = Path(config.fallback_embeddings_path) если задан иначе ""
  Если fb_path пустой или not fb_path.exists():
    WARN: "fallback_embeddings.npy не найден. Запустите scripts/build_fallback"
  Иначе:
    import numpy as np; d = np.load(str(fb_path), allow_pickle=True).item()
    OK: f"{len(d)} слов в fallback-словаре"
  При Exception: FAIL.

Функция check_synonyms(config) -> CheckResult:
  Если not Path(config.synonyms_path).exists(): FAIL
  data = json.loads(Path(config.synonyms_path).read_text(encoding="utf-8"))
  Если not isinstance(data, dict): FAIL "Неверный формат synonyms.json"
  OK: f"{len(data)} слов"
  Если len(data) == 0: WARN "Пустой словарь. Запустите scripts/build_synonyms"

Функция check_domain_templates(config) -> CheckResult:
  Аналогично проверить domain_templates.json и domain_keywords.json.
  OK: f"{len(data)} доменов"
  Если файл отсутствует: FAIL.

Функция check_faiss(config) -> CheckResult:
  Если not config.use_faiss:
    OK: "use_faiss=false (FAISS не используется)"
    return
  try: import faiss; OK "faiss-cpu установлен"
  except ImportError: WARN "use_faiss=true но faiss-cpu не установлен"
  faiss_path = Path(config.faiss_index_path) если задан
  Если not faiss_path.exists(): WARN "FAISS индекс не найден. Запустите build_faiss"

Функция check_fastapi(config) -> CheckResult:
  try: import fastapi, uvicorn; OK "fastapi + uvicorn установлены"
  except ImportError as e: WARN f"REST API недоступен: {e}"

Функция check_prometheus(config) -> CheckResult:
  Если not config.use_metrics:
    OK: "use_metrics=false (метрики отключены)"
    return
  try: import prometheus_client; OK "prometheus_client установлен"
  except ImportError: WARN "use_metrics=true но prometheus_client не установлен"

Функция check_logs_dir() -> CheckResult:
  logs_dir = Path("logs")
  Если not logs_dir.exists(): WARN "Папка logs/ не существует. Запустите setup_project.py"
  Если не writable (попытаться создать временный файл): FAIL "logs/ не доступна для записи"
  OK: "logs/ доступна"

Функция run_all_checks(config_path: Path) -> list[CheckResult]:
  Сначала загрузить конфиг.
  Если конфиг не загрузился — остальные проверки пропустить (часть из них упадёт).
  Вернуть список результатов всех проверок.

Функция print_report(results: list[CheckResult]) -> bool:
  Вывести таблицу:
    Компонент              | Статус | Детали
    -----------------------|--------|---------------------------
    config.json            | OK     | db_path=..., min_conf=0.3
    База знаний (SQLite)   | OK     | 3 концепта
    FastText модель        | WARN   | Не найдена: models/cc.ru.300.bin
    fallback_embeddings    | WARN   | Не найден. Запустите build_fallback
    synonyms.json          | OK     | 20 слов
    domain_templates.json  | OK     | 2 домена
    FAISS                  | OK     | use_faiss=false
    FastAPI / uvicorn      | WARN   | REST API недоступен: No module named 'fastapi'
    Prometheus             | OK     | use_metrics=false
    Директория logs/       | OK     | logs/ доступна

  Легенда: ✓ OK   ⚠ WARN   ✗ FAIL
  Итог: "Критичных ошибок: N | Предупреждений: M"
  Вернуть: True если нет FAIL (только OK и WARN).

Блок if __name__ == "__main__":
  Условный sys.path.insert.
  argparse: --config (default="configs/config.json"), --strict (флаг: WARN тоже = ошибка).
  results = run_all_checks(args.config)
  ok = print_report(results)
  Если args.strict: ok = ok and all(r.status != "WARN" for r in results)
  sys.exit(0 if ok else 1)

ИНТЕРФЕЙС
----------
  run_all_checks(config_path: Path) -> list[CheckResult]
  print_report(results: list[CheckResult]) -> bool

ПРОВЕРКА РАБОТОСПОСОБНОСТИ
--------------------------

  Ожидаемый вывод: таблица со статусами всех компонентов.
  Если все критичные OK: exit code 0.

  python -m scripts.healthcheck --strict
  Ожидаемый вывод: тот же + exit code 1 если есть WARN.

  # Автоматическая проверка в CI:
  python -m scripts.healthcheck && echo "Проект готов к запуску"

ДЕТАЛИ ОБНОВЛЕНИЯ main.py (_setup_logging)
-------------------------------------------
Текущая реализация _setup_logging() — только StreamHandler (консоль).
Обновить: добавить RotatingFileHandler если папка logs/ существует и доступна.

Новая логика _setup_logging(log_level: str = "INFO") -> logging.Logger:
  1. Определить LOG_FORMAT:
       "[%(asctime)s] %(levelname)s %(name)s: %(message)s"
       date_format: "%Y-%m-%d %H:%M:%S"

  2. Создать корневой логгер (или logger для пакета "ai_terminator"):
       logger = logging.getLogger("ai_terminator")
       logger.setLevel(getattr(logging, log_level.upper(), logging.INFO))

  3. Если logger.handlers уже есть — не добавлять дублирующих хэндлеров.
     (Защита от повторного вызова _setup_logging при повторном импорте.)

  4. StreamHandler (консоль):
       sh = logging.StreamHandler(sys.stdout)
       sh.setFormatter(logging.Formatter(LOG_FORMAT, date_format))
       logger.addHandler(sh)

  5. RotatingFileHandler (файл):
       logs_dir = LOG_FILE.parent  (LOG_FILE = PROJECT_ROOT / "logs" / "ai_terminator.log")
       Если logs_dir.exists() and os.access(str(logs_dir), os.W_OK):
         try:
           from logging.handlers import RotatingFileHandler
           fh = RotatingFileHandler(
               str(LOG_FILE),
               maxBytes=5 * 1024 * 1024,   (5 МБ)
               backupCount=3,               (хранить 3 архивных файла)
               encoding="utf-8",
           )
           fh.setFormatter(logging.Formatter(LOG_FORMAT, date_format))
           logger.addHandler(fh)
           logger.debug("Логирование в файл: %s", LOG_FILE)
         except (OSError, PermissionError) as exc:
           logger.warning("Не удалось создать файловый лог: %s", exc)
       Иначе:
         logger.debug("logs/ недоступна — используется только консоль")

  6. return logger

Создать Makefile (Linux/macOS) и make.bat (Windows) с удобными командами
для типичных операций разработки: setup, test, run, lint, eval, clean.

ПОЧЕМУ ЭТО НУЖНО
----------------
Без Makefile разработчик должен помнить (или искать в документации) длинные
команды: python -m scripts.setup_all, python -m pytest tests/ -v --tb=short,

слова. Для Windows Batch-файл make.bat эмулирует базовые цели Makefile.

ДЕТАЛИ РЕАЛИЗАЦИИ Makefile (для Linux/macOS)
----------------------------------------------
  .PHONY: setup test run api eval profile health clean lint help

  help:                     — показать все доступные цели
  setup:                    — setup_project.py + pip install + scripts/setup_all
  test:                     — python -m pytest tests/ -v --tb=short
  test-fast:                — pytest -x (остановиться на первой ошибке)
  run:                      — python main.py (с примером запроса)
  api:                      — python -m scripts.run_api
  eval:                     — python -m scripts.evaluate
  profile:                  — python -m scripts.profile

ДЕТАЛИ РЕАЛИЗАЦИИ make.bat (для Windows)
------------------------------------------
  Windows не понимает Makefile. make.bat — Batch-скрипт с командами через IF.

  Структура:
    @echo off
    IF "%1"==""         GOTO help
    IF "%1"=="help"     GOTO help
    IF "%1"=="setup"    GOTO setup
    IF "%1"=="test"     GOTO test
    IF "%1"=="run"      GOTO run
    IF "%1"=="api"      GOTO api
    IF "%1"=="health"   GOTO health
    IF "%1"=="clean"    GOTO clean
    IF "%1"=="eval"     GOTO eval
    IF "%1"=="profile"  GOTO profile
    GOTO unknown

    :help
    echo Использование: make.bat [цель]
    echo Доступные цели: setup test test-fast run api health eval profile
    echo                 build-fallback build-faiss build-synonyms
    echo                 build-centroids export-kb clean lint
    GOTO end

    :setup
    python setup_project.py
    pip install -r requirements.txt
    python -m scripts.setup_all
    GOTO end

    :test
    python -m pytest tests/ -v --tb=short
    GOTO end

    :run
    python main.py --input "{"term":"ключ","hints":["техника"]}"
    GOTO end

    :health
    python -m scripts.healthcheck
    GOTO end

    :clean
    FOR /D /R . %%d IN (__pycache__) DO @IF EXIST "%%d" RD /S /Q "%%d"
    FOR /D /R . %%d IN (.pytest_cache) DO @IF EXIST "%%d" RD /S /Q "%%d"
    IF EXIST logs\*.log DEL /Q logs\*.log
    echo Очистка завершена.
    GOTO end

    :unknown
    echo Неизвестная цель: %1. Выполните make.bat help
    EXIT /B 1

    :end
    EXIT /B 0

  Запуск: make.bat test  (аналог make test на Linux)

КРАЙНИЕ СЛУЧАИ:
  - Makefile: если python не в PATH — явно задать PYTHON=python3 или путь.
  - make.bat: аргументы с пробелами — оборачивать в двойные кавычки.
  - В CI (GitHub Actions) использовать python -m pytest напрямую, не make.
